
# rehype-easytex

Use `$...$` and `$$...$$` directly inside HTML, without manually writing KaTeX wrapper elements.

`rehype-easytex` is a small **rehype plugin** that lets you write inline and block math naturally inside text nodes. It converts those expressions into the structure expected by KaTeX, so they work seamlessly with `rehype-katex`.

---

## тЬи Features

* Write inline math with `$...$`
* Write display math with `$$...$$`
* Works anywhere text nodes are valid in HTML
* Designed to integrate directly with `rehype-katex`
* No need to manually write `<span class="math-inline">` or `<div class="math-display">`

## ЁЯЪА Usage

```ts
import { unified } from 'unified'
import rehypeParse from 'rehype-parse'
import rehypeStringify from 'rehype-stringify'
import rehypeKatex from 'rehype-katex'
import rehypeDocument from 'rehype-document'
import rehypeEasytex from 'rehype-easytex'
import { read, write } from "to-vfile"

const file = await unified()
  .use(rehypeParse, { fragment: true })
  .use(rehypeEasytex)
  .use(rehypeKatex)
  .use(rehypeDocument, {
    css: 'https://cdn.jsdelivr.net/npm/katex@0.16.8/dist/katex.min.css'
  })
  .use(rehypeStringify)
  .process(await read('./input.html'))

file.path = "./output.html"
await write(file)
```

---

## ЁЯза How it works

`rehype-easytex` scans **text nodes** in the HAST (HTML AST) and:

1. Detects math expressions delimited by:

   * `$...$` (inline)
   * `$$...$$` (display)

2. Splits the original text node into multiple nodes:

   * plain text segments
   * math nodes compatible with `rehype-katex`

3. Leaves the rest of the tree unchanged

This allows `rehype-katex` to process math expressions as if they were written in its expected format.

## ЁЯзй Example

### Input

```html
<p>
  This is inline math: $a^2 + b^2 = c^2$
</p>

<p>
  And a block:
  $$\int_0^1 x^2 dx$$
</p>
```

---

### Output (simplified)

```html
<p>
  This is inline math:
  <span class="math-inline">...</span>
</p>

<p>
  And a block:
  <div class="math-display">...</div>
</p>
```

(The actual output is generated by KaTeX.)

## тЪая╕П Notes

* Must be used **before** `rehype-katex`
* Only processes content inside **text nodes**
* Does not parse math inside HTML attributes


---

## ЁЯз▒ Pipeline order

Correct order:

```ts
.use(rehypeParse)
.use(rehypeEasytex)   // ЁЯСИ must come before katex
.use(rehypeKatex)
.use(rehypeStringify)
```
